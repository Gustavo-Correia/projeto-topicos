Single-package Flutter app rooted at `lib/main.dart`, bootstrapping `Hive.initFlutter` and two boxes (`subscriptions_box`, `settings_box`) before launching `MaterialApp`. State is centralized in two `ChangeNotifier`s registered via `MultiProvider`: `SubscriptionProvider` (domain logic for CRUD, filtering, sorting, aggregations like `totalMonthly`/`categoryTotals`) and `SettingsProvider` (user profile, onboarding flag, theme selection). Providers depend on abstracted storage interfaces (`SubscriptionStorage`, `SettingsStorage`) implemented by `*_storage_service.dart` classes backed by Hive JSON serialization; this lets tests inject fakes. Domain models live under `models/` (`Subscription`, `AppSettings`) as immutable value objects with `copyWith` + `toMap`/`fromMap` converters. Screens under `screens/` are stateless/stateful entry points that read providers through `context.read`/`Consumer`; the splash screen orchestrates parallel initialization of settings and subscriptions then routes to `HomeScreen` or `OnboardingScreen`. Theming is decoupled into `utils/app_colors.dart` where `AppTheme` maps to a `ThemeData` and `AppColors.of(context)` resolves the active theme from `SettingsProvider`. Reusable UI fragments sit in `widgets/`. Dependency direction: screens → providers → services → models/utils; providers never import screens.
