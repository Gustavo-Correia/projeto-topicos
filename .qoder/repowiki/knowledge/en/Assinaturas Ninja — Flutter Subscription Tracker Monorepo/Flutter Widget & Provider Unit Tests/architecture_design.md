Flat test suite under `test/` with one file per production unit: `subscription_model_test.dart` exercises pure `Subscription` domain logic (due-date math, serialization), `settings_provider_test.dart` and `subscription_provider_test.dart` drive stateful `ChangeNotifierProvider`s via constructor-injected storage interfaces (`SettingsStorage`, `SubscriptionStorage`) implemented by in-memory fakes defined locally in each file. `onboarding_screen_test.dart` and `widgets_test.dart` are integration-style `testWidgets` that mount a `MaterialApp` with `MultiProvider`/`ChangeNotifierProvider` to assert rendered text and user gestures (drag/tap). Dependency direction is strictly downward — tests import from `package:assinaturas_ninja/models`, `providers`, `services`, and `widgets`; no cross-test imports exist.
