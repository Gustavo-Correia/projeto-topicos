App Android de activity única construído com o plugin Gradle do Flutter (`dev.flutter.flutter-gradle-plugin`) em `app/build.gradle.kts`; `MainActivity` estende `io.flutter.embedding.android.FlutterActivity`, então toda a lógica de UI vive na camada Dart/Flutter acima. O manifesto registra `.MainActivity` como activity LAUNCHER exportada com modo de lançamento `singleTop` e encaminha para o NormalTheme do Flutter via entrada meta-data; `flutterEmbedding=2` habilita o embedding v2. Build targeta Java/Kotlin 17, obtém compileSdk/ndkVersion/minSdk/versionCode/versionName do tooling Flutter via propriedades `flutter.*`, e aponta `flutter { source = "../.." }` para a raiz do repositório onde o código Dart compartilhado reside. Recursos em `src/main/res/` fornecem apenas o background de lançamento padrão e ícones mipmap — nenhum código de negócio Kotlin/Java customizado está presente além do `GeneratedPluginRegistrant.java` gerado.
