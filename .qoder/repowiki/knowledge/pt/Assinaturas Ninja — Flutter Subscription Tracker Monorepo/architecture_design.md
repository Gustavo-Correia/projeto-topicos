O repositório é um projeto Flutter padrão enraizado na raiz do workspace: `pubspec.yaml` declara todas as dependências Dart compartilhadas (Provider, Hive, intl, uuid) consumidas pelo app Flutter (`lib/`) e sua suíte de testes (`test/`). Os diretórios `android/` e `ios/` são os shells de embedding nativo gerados pelo Flutter; não contêm lógica de negócio e existem apenas para iniciar o mesmo entry point Dart. `docs/` contém referências de arquitetura, UI, validação e tarefas que tanto desenvolvedores quanto agentes de IA consultam durante o desenvolvimento. Não há conexão entre os filhos além do fluxo único `flutter run` / `flutter build` que inicializa o engine Flutter dentro de cada shell de plataforma.
